# 中国合规白皮书 (China Compliance Whitepaper)

**版本**: v1.0  
**生效日期**: 2026-07-12  
**适用范围**:waibao 在中国大陆境内开展的全部业务

---

本文件阐述 waibao 平台在中国大陆地区运营时的合规策略、技术达标路径与差距分析,涵盖《个人信息保护法》(PIPL)、《数据安全法》(DSL)、《网络安全法》(CSL) 与网络安全等级保护 2.0(MLPS 2.0)三级要求。

## 第一部分 — 个人信息保护法(PIPL)对照表

PIPL 自 2021 年 11 月 1 日起施行,被称为中国版 GDPR。我们逐条对照 waibao 落地情况。

| PIPL 条款 | 核心要求 | waibao 现状 | 技术 / 流程 |
|---|---|---|---|
| **第 13 条** | 处理个人信息的合法基础 | 合同必要 / 明示同意 / 法定义务 / 法定职责 | 见《隐私政策》§2 |
| **第 14 条** | 同意应"知情、自愿、明确" | Cookie 横幅提供独立同意;跨境同意单独勾选 | `ConsentService.record_consent` |
| **第 15 条** | 处理敏感信息需"单独同意" | 实名认证、身份证号、银行账号均弹窗单独授权 | `services/pii_field_encryption.py` |
| **第 17 条** | 处理前告知 | 隐私政策、Cookie 政策、ToS 三语完整版 | `/legal/{terms,privacy,cookies,dpa}` |
| **第 23 条** | 自动化决策需透明 + 结果公平 | AI 匹配结果可解释;提供"关闭 AI 辅助"开关 | `MatchingMatchExplain` |
| **第 24 条** | 自动化决策可拒绝 | 隐私设置关闭 AI 匹配 | `account/privacy` |
| **第 27 条** | 公开个人信息合理范围 | 简历公开字段默认最小化;企业 HR 需明示同意才可查看完整简历 | 候选人默认公开:姓名 + 邮箱 + 工作年限 |
| **第 38 条** | 跨境传输需满足三选一 | 默认境内存储;跨境需标准合同 / 安全评估 / 认证 | `data_residency.py` |
| **第 39 条** | 跨境传输告知 + 单独同意 | Cookie 横幅 + 隐私设置均可单独控制 | `cross_border` consent category |
| **第 40 条** | 关键信息基础设施运营者需境内存储 | waibao 不属于 CIIO,但默认境内存储 | `Region.CN` 默认 |
| **第 44 条** | 数据主体权利 | 知情 / 访问 / 更正 / 删除 / 可携 / 撤回 | `/api/gdpr/*` |
| **第 45 条** | 配合监管 | DPO 邮箱公开;年度合规报告 | `dpo@waibao.example` |
| **第 47 条** | 合理响应权利请求 | 15 工作日内;特殊情况 30 天 | 见《隐私政策》§5 |
| **第 50 条** | 处理者义务 | DPA 模板 + 分处理者清单 | `DATA_PROCESSING_AGREEMENT.md` |
| **第 51 条** | PIPIA(个人信息保护影响评估) | 跨境前 / 自动化决策上线前 / 业务重大变更前 | `services/pipia.py` 计划 |
| **第 52 条** | 个人信息泄露通知 | 24 小时内通知用户 + 监管 | `services/incident_response.py` |
| **第 73 条** | 大型互联网平台义务 | waibao 用户规模较小,暂不适用 | 不适用 |

### PIPL 差距清单
- [ ] **PIPIA 工具自动化**:影响评估表支持在线填写 + 自动归档(计划 T1203)
- [ ] **法定代表人 / DPO 任命文件**:已起草,需要正式任命(已落实)
- [ ] **个人信息处理记录(Art.30 内部版)**:audit_log 已支持 GDPR Art.30 字段,需扩展至 PIPL 字段集(进行中)

## 第二部分 — 数据安全法(DSL)要求清单

DSL 自 2021 年 9 月 1 日起施行,聚焦数据全生命周期安全。

| DSL 条款 | 要求 | waibao 现状 |
|---|---|---|
| **第 21 条** | 数据分类分级 | PII / 一般数据 / 公开数据三档,见附录 |
| **第 27 条** | 数据安全应急响应 | 7×24 值班;事件确认 24h 内通知 |
| **第 32 条** | 收集合法、正当、必要 | 最小化原则,见《隐私政策》§1 |
| **第 33 条** | 重要数据处理者义务 | waibao 当前不属于"重要数据处理者",但仍按其标准建设 |
| **第 36 条** | 境外司法机关要求提供数据须经主管机关批准 | 已写入 DPA 第 8.4 条 |
| **第 39 条** | 国家数据安全审查 | LLM 调用、跨境功能上线前进行内部审查 |

### DSL 数据分类分级矩阵

| 级别 | 类别 | 字段示例 | 加密 | 访问 |
|---|---|---|---|---|
| **L4 核心** | 实名认证 / 身份证号 | id_card_no | Fernet 字段级 + 单独密钥 | 仅 KYC 服务 + DPO |
| **L3 重要** | 简历 / 工作经历 / 银行账号 | resume_text, bank_card | Fernet 字段级 | RBAC + MFA |
| **L2 一般** | 邮箱 / 手机 / 姓名 | email, phone, full_name | Fernet 字段级 | RBAC |
| **L1 公开** | 公司名 / 公开技能 | company_name, headline | 不加密 | 公开 |

## 第三部分 — 网络安全法(CSL)要求清单

CSL 自 2017 年 6 月 1 日起施行,是中国网络安全的基础性法律。

| CSL 条款 | 要求 | waibao 现状 |
|---|---|---|
| **第 21 条** | 等级保护 | 已按等保 2.0 三级建设(见第四部分) |
| **第 22 条** | 网络产品和服务安全 | 自研 + 第三方均经过安全评估 |
| **第 24 条** | 实名认证 | 中国大陆用户登录需手机号 + 验证码 |
| **第 27 条** | 禁止危害网络安全行为 | 内嵌 XSS / SQL 注入防护;CSP 头 |
| **第 28 条** | 协助公安机关 | 已制定协助流程(应急响应预案) |
| **第 37 条** | 关键信息基础设施数据境内存储 | waibao 默认境内存储 |
| **第 41 条** | 个人信息保护 | 与 PIPL 一致 |

## 第四部分 — 网络安全等级保护 2.0(MLPS 2.0)三级差距分析

### 4.1 MLPS 2.0 三级概览

三级是**重要行业 / 关键信息基础设施**的标准。涵盖物理安全、网络安全、主机安全、应用安全、数据安全、安全管理制度、安全管理机构、安全管理人员、安全建设管理、安全运维管理十个方面,共约 **211 个测评项**。

## 第五部分 — 实名认证集成方案

### 5.1 背景

根据《网络安全法》第 24 条与《互联网用户公众账号信息服务管理规定》,互联网平台需对用户进行实名认证。waibao 在中国大陆境内服务时,采用以下三种 OAuth 实名认证通道之一。

### 5.2 钉钉 OAuth 认证

| 步骤 | 详情 |
|---|---|
| **1. 应用创建** | 在钉钉开放平台(open-dev.dingtalk.com)创建"企业内部应用" |
| **2. 权限申请** | `Contact.User.mobile` / `Contact.User.email` / `Contact.User.name` / `Contact.User.id_card`(可选) |
| **3. OAuth 流程** | 前端 → 后端 → 钉钉授权页 → 用户确认 → 回调 → 换取 access_token |
| **4. 信息存储** | 仅保留:手机号(Fernet 加密)、姓名(Fernet 加密)、钉钉 unionid |
| **5. 原始数据** | 身份证号原始影像在认证完成后 **24 小时内物理删除** |
| **6. 审计** | 每次认证写 audit_log:`action=kyc`, `resource=user`, `provider=dingtalk` |

### 5.3 企业微信 OAuth 认证

| 步骤 | 详情 |
|---|---|
| **1. 应用创建** | 企业微信管理后台 → "应用管理" → 创建自建应用 |
| **2. 权限申请** | `userid` / `mobile` / `email` / `name` |
| **3. OAuth 流程** | 同上 |
| **4. 信息存储** | 同上 |
| **5. 审计** | `provider=wechat_work` |

### 5.4 飞书 OAuth 认证

| 步骤 | 详情 |
|---|---|
| **1. 应用创建** | 飞书开放平台(open.feishu.cn) → 创建企业自建应用 |
| **2. 权限申请** | `contact:user.id:readonly` / `contact:user.phone:readonly` / `contact:user.email:readonly` |
| **3. OAuth 流程** | 同上 |
| **4. 信息存储** | 同上 |
| **5. 审计** | `provider=feishu` |

### 5.5 兜底 — 手机号 + 验证码

对于未绑定上述任一渠道的用户,提供手机号 + 短信验证码实名方式。该方式视为"低强度实名",仅适用于轻度使用场景;关键功能(如签约、开票)仍需绑定上述 OAuth 之一。

### 5.6 实名数据保留

依据 PIPL 第 19 条"实现处理目的后应及时删除",waibao 的实名信息保留策略:

- **手机号 / 邮箱 / 姓名**:账户存续期间持续保留,用于登录与通知
- **身份证号原始影像**:**24 小时内物理删除**
- **认证日志(去标识)**:保留 5 年(对应法律义务)
