# 数据处理协议 (Data Processing Agreement / DPA)

**版本**: v1.0  
**生效日期**: 2026-07-12  
**适用于**:waibao B2B 客户与 waibao Technology Co., Ltd. 之间的数据处理关系  
**相关法律**:GDPR Art. 28、《个人信息保护法》第 21 条

---

本 DPA 是 waibao 服务条款的组成部分,规范客户(数据"控制者")委托 waibao(数据"处理者")处理个人数据的权利、义务与责任。

## 一、定义

1.1 **"控制者"(Controller)**:决定数据处理目的和方式的企业或组织,即本协议项下使用本平台的客户。  
1.2 **"处理者"(Processor)**:代表控制者处理个人数据的实体,即 waibao Technology Co., Ltd.。  
1.3 **"分处理者"(Sub-processor)**:处理者委托处理个人数据的第三方,详见附录 A。  
1.4 **"客户数据"(Customer Data)**:客户通过本平台提交、存储或处理的全部个人数据。  
1.5 **"数据主体"(Data Subject)**:客户数据所涉及的自然人,如候选人、HR 等。  
1.6 **"适用法律"**:GDPR、《个人信息保护法》、UK GDPR、CCPA/CPRA 及各方所在司法辖区的数据保护法。

## 二、处理范围

| 项目 | 内容 |
|---|---|
| 处理目的 | 提供简历解析、AI 匹配、多人协同、合规审计等本平台核心服务 |
| 数据类别 | 候选人个人信息、HR 账户信息、协作内容、日志 |
| 数据主体类别 | 求职者(候选人)、招聘方(雇主 HR)、用人单位决策人 |
| 处理方式 | 收集、存储、解析、加密、传输、删除(自动化与人工混合) |
| 保留期限 | 详见《隐私政策》第 3.3 节 |
| 处理地点 | 中国大陆(SG / US 可选,需经单独同意) |

## 三、处理者义务

3.1 **仅按指示处理**:waibao 仅根据客户书面(或系统配置)指示处理客户数据,不得超出授权范围。  
3.2 **保密性**:对接触客户数据的人员施加严格的保密义务,MFA + 最小权限。  
3.3 **安全措施**:实施本协议第六条所列技术与组织措施(TOMs),防止数据泄露、损毁或未经授权访问。  
3.4 **分处理者管理**:
- 事前披露所有分处理者清单(附录 A)
- 任何新增分处理者,提前 30 天书面(或站内通知)告知客户
- 与分处理者签署不低于本协议标准的 DPA

3.5 **协助义务**:协助客户履行数据主体权利请求、监管沟通、安全事件通知等义务。  
3.6 **审计权**:客户在提前 30 天书面通知后,有权每年一次(或重大事件后)对 waibao 进行审计;waibao 可提供独立第三方审计报告替代。  
3.7 **数据返回与删除**:服务终止后 30 天内,按客户选择:
- (a) 通过平台导出工具返回全部客户数据;且
- (b) 在客户确认收到后 30 天内物理删除全部客户数据,除非法律要求继续保留

## 四、控制者义务

4.1 **合法性基础**:确保有合法基础收集客户数据并委托 waibao 处理。  
4.2 **数据主体告知**:已在隐私政策或同等文件中告知数据主体其数据将被本平台处理。  
4.3 **跨境传输**:若客户数据涉及跨境传输,客户须事先获得数据主体单独同意(参见《Cookie 政策》中的"cross_border"分类)。  
4.4 **数据主体请求**:收到数据主体请求后,在适用法律规定的时限内联系 waibao 协助处理。

## 五、数据主体权利

5.1 访问、更正、删除、可携、限制处理、反对自动化决策等权利。  
5.2 waibao 提供以下 API 协助控制者履行:
- `GET /api/gdpr/export` — 数据可携(导出)
- `DELETE /api/gdpr/all-data` — 被遗忘权(删除)
- `POST /api/gdpr/consent` — 同意记录与撤回
- `GET /api/audit/personal` — 个人访问审计记录

## 六、技术与组织措施(TOMs)

| 类别 | 措施 |
|---|---|
| 加密 | PII 字段级 Fernet (AES-128-CBC + HMAC-SHA256);传输 TLS 1.3 |
| 密钥管理 | 环境变量注入 + 定期轮转;HSM 可选 |
| 访问控制 | RBAC + MFA + IP 白名单;最小权限 |
| 日志与审计 | 所有 PII 访问记录到 audit_log,保留 6 个月(关键岗位 5 年) |
| 备份 | 每日全量 + 6 小时增量;异地多活(中国大陆境内) |
| 漏洞管理 | 季度第三方渗透测试;月度依赖扫描 |
| 事件响应 | 7×24 小时值班,事件确认后 24 小时内通知 |
| 业务连续性 | RPO ≤ 1 小时,RTO ≤ 4 小时 |
| 物理安全 | 数据中心通过等保 2.0 三级、ISO 27001、SOC 2 Type II |
| 人员管理 | 入职背调 + 保密协议 + 定期培训 |

## 八、跨境传输机制

8.1 **中国大陆 → 海外**:依据《个人信息保护法》第 38 条 + 第 40 条,采用"标准合同 + 安全评估"双轨制;  
8.2 **EU → 第三国**:GDPR Art. 46 标准合同条款(SCCs);  
8.3 **UK → 第三国**:UK International Data Transfer Addendum;  
8.4 **传输前**:必填 PIPIA / TIA 评估表,留存不少于 3 年。

## 九、责任与赔偿

9.1 任一方因违反本协议导致数据主体或第三方损失的,由违约方承担赔偿责任。  
9.2 waibao 的总责任以最近 12 个月客户支付的费用为上限,除非适用法律禁止此类限制。  
9.3 双方应及时相互通知可能涉及本协议项下数据的事件。

## 十、期限与终止

10.1 本协议自客户首次使用本平台之日起生效,至客户账户终止且数据全部删除后失效。  
10.2 即使本协议终止,第 5、6、9 条仍继续有效。

## 十一、签署

本 DPA 通过电子方式签署(平台订单确认页面勾选 + 站内记录)。客户首次使用本平台即视为已签署。
