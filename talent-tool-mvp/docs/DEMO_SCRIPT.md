# 演示流程 (DEMO_SCRIPT)

> v11.0 / T6113 — 甲方验收 / 产品演示的标准流程,覆盖 5 个核心业务场景。
> 前置:已完成一键本地部署(`bash scripts/start_local.sh`)并导入测试数据
> (`python scripts/seed_test_data.py --with-sample-resumes`)。
> 演示入口:http://localhost:3000

**演示原则**:
- 每个场景有明确角色(求职者 / HR / 老板 / 部门负责人);
- 强调“数据不出甲方环境”——每步可现场抓包验证 LLM/OCR 走本地;
- 每个场景结尾给出“验收看点”,对应《验收清单》编号。

---

## 场景 1:求职者注册 → 上传简历 → AI 聊天 → 生成画像

**角色**:求职者(以演示账号或新注册)
**对应验收**:AC-05 / AC-06 / AC-07 / AC-08

**步骤**:
1. 打开 http://localhost:3000 → 点击“求职者入口” → 注册(手机/邮箱)→ 登录;
2. 进入“我的简历” → 上传一份简历(用 `scripts/seed_resumes/sample_resume_zh.txt` 转出的文件,或演示图片);
3. 系统调用**本地 PaddleOCR** 解析 → 展示结构化字段(姓名/技能/学历/经历);
4. 进入“AI 聊天” → 提问“根据我的简历,我适合体育用品行业哪些岗位?” → AI(本地 Ollama)回复;
5. 进入“人才画像” → 点击“生成画像” → 展示技能标签、核心优势、推荐方向 → 求职者确认/修正。

**验收看点**:
- 简历解析字段准确(OCR 本地完成,抓包无出网);
- AI 对话连贯、贴合简历内容;
- 画像结构化、可读、可修正。

## 场景 2:企业注册 → 上传资质 → 创建岗位 → AI 细化

**角色**:企业 HR / 管理员
**对应验收**:AC-10 / AC-11 / AC-12 / AC-13

**步骤**:
1. 首页 → “企业入口” → 注册企业(公司名/行业/规模)→ 登录企业管理后台;
2. 进入“企业资质” → 上传营业执照图片 → 系统**本地 OCR** 识别 + AI 真实性初筛 → 标记审核状态;
3. 进入“岗位管理” → “新建岗位” → 填写标题/描述/技能要求/薪资/地点 → 保存;
4. 在岗位编辑页点击“AI 细化 JD” → AI 输出营销化、更专业的 JD 文案 → 一键采纳;
5. (可选)查看资质检测标记,演示对可疑资质的人工复核流程。

**验收看点**:
- 资质 OCR 本地识别、存档;
- JD 经 AI 细化后明显更专业、有吸引力;
- 假资质被标记供复核。

## 场景 3:匹配 → 推送 → 企业查看 → 联系 → 面试

**角色**:HR(企业侧)
**对应验收**:AC-15 / AC-17 / AC-24

**步骤**:
1. 在企业管理后台进入某岗位详情 → “推荐候选人” → 展示按综合分排序的候选人列表;
2. 点开 Top 候选人 → 查看匹配分数、技能重叠、优势/差距、AI 解释;
3. 对心仪候选人点击“发起联系” → 填写联系内容 → 系统记录**联系日志**;
4. 点击“安排面试” → 选时间/方式 → 生成**面试日程**,多方可见;
5. 进入“智能推荐”位 → 查看系统主动推荐的人才与岗位(对应 AC-24)。

**验收看点**:
- 匹配有分数 + 解释,可解释性强;
- 联系日志落库、面试日程生成;
- 推荐结果与岗位上下文相关。

## 场景 4:情绪 → 转人工

**角色**:求职者 + 人工坐席/HR
**对应验收**:AC-18 / AC-19

**步骤**:
1. 求职者在 AI 聊天中表达负面情绪(如“找工作找得很焦虑,感觉没希望”)→ AI 识别情绪 → 给出**温和、共情的关怀回应**;
2. 求职者进一步表达自伤倾向(演示话术,如“我不想活了”)或劳动争议信号(如“公司不给我发工资,我要去维权但很绝望”)→ 系统**触发转人工规则**;
3. 系统立即:暂停纯 AI 应对 → 通知人工坐席/HR → 标记工单为高优先级 → 保留完整对话记录;
4. 人工坐席在后台看到告警 → 接管对话 → 进行人工介入。

**验收看点**:
- 普通负面情绪 → AI 关怀(不误触发转人工);
- 自伤风险 / 劳动争议信号 → 准确触发转人工 + 告警;
- 转人工后对话记录完整、可追溯。

## 场景 5:多方协作

**角色**:求职者 + HR + 部门负责人
**对应验收**:AC-20 / AC-21 / AC-22 / AC-26

**步骤**:
1. 围绕一位候选人创建**协作房间**,邀请 HR 与用人部门负责人加入;
2. 三方在房间内实时沟通(消息带角色标识),讨论是否推进;
3. 制造一方“沉默”(长时间不回复)→ 系统**沉默激活**提醒 → 关键事件**多方通知**;
4. 房间内展示**共识度评分**:随各方反馈(赞同/保留/反对)实时更新;
5. HR 使用**HR 助手**:勾选 2-3 位候选人 → 对比 → 生成评估报告 → 一键生成面试题。

**验收看点**:
- 多方实时协作顺畅,角色清晰;
- 沉默激活与通知生效;
- 共识度随反馈更新、可视化;
- HR 助手对比/报告/面试题内容合理。
