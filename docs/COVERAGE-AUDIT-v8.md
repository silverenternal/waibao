## 维度 1: 16 项甲方需求 - 深度评估

虽然 v1-v7 都已 16/16 ✅,**但"功能有"≠"做透"**。甲方要的是"智能体成为求职者的朋友/HR 的得力助手",这需要深度体验:

### 求职者侧 (1.x)

| 编号 | 评级 | v7.0 现状 | "做透" 还差什么 |
|---|---|---|---|
| **1.1 智能/知心朋友 + 学历上传** | 🟡 | 简历上传 + OCR + 视频简历 + ProfileCard | ProfileCard 是"数据展示"不是"知心朋友"。缺:**情感化对话** + **主动关心**(生日/职业空窗期提醒) + **私聊风格个性化** |
| **1.2 工作状态 + 评价** | 🟡 | 日报 + 语音 + 趋势图 + 行动项 | 评价是模板化,缺:**针对具体行业/职位的专业评价** + **行动项完成度跟踪** + **建议的可执行性** |
| **1.3 频繁互动** | 🟡 | Realtime + 协同房间 | "频繁"是响应快,**但缺主动**: 智能体不会主动 push("今天状态如何?") |
| **1.4 喜怒哀乐** | 🟡 | 折线图 + 高风险告警 | 告警有,但**关怀动作缺**: 心情差时智能体主动温暖对话,HR 自动关怀 |
| **1.5 澄清 → 画像 + 需求** | 🟡 | 画像 + 追问 + 冲突 | 画像生成有,**但"求职者确认"机制缺**: 用户看不到"我理解的你"是否准确 |
| **1.6 职业规划师** | 🟡 | 真实市场 + 学习资源 + 面试准备 | 规划生成有,**但执行跟踪弱**: 计划"3 个月学 X"是否真执行?没机制 |

### 用人单位侧 (2.x)

| 编号 | 评级 | v7.0 现状 | "做透" 还差什么 |
|---|---|---|---|
| **2.1 真诚 HR + 老板助手** | 🟡 | Persona + 老板偏好 + 升级 | 偏好存储有,**但"真诚"难量化**: 模板化回复?语气个性化?老板是否真觉得"懂我"? |
| **2.2 资质验证** | 🟡 | OCR + 工商 + 信用代码 | 基础验证有,**但假资质鉴别弱**: 假 PS 营业执照?过期证件? |
| **2.3 愿景战略传达** | 🟡 | Vision + 4 层战略地图 | 战略地图有,**但"传达"缺**: 战略更新如何让所有 HR/候选人看到?战略对招聘的影响? |
| **2.4 老板描述人才框架** | 🟡 | TalentBrief + 偏见检测 | 偏见检测有,**但"纠正"弱**: 老板看不到替代话术?不应用? |
| **2.5 部门负责人细化 JD** | 🟡 | JobSpec + 模板 + over-spec | 模板+over-spec 有,**但 JD 撰写质量**?好的 JD 是 marketing + 招聘的混合产物 |
| **2.6 规章制度** | 🟡 | Policy + 浏览 + 搜索 | 浏览/搜索有,**但求职者真懂吗?** 缺"AI 解释制度"功能 |
| **2.7 多方互动** | 🟡 | 协同房间 + @mention | 房间有,**但"频繁"不够**: 没人发新消息就沉默了,缺 notification 触达 |
| **2.8 用人方信息澄清** | 🟡 | 画像 + StakeholderMatrix | 画像有,**但共识度算法简单** |
| **2.9 成为 HR** | 🟡 | 工单 + SLA | 被动响应有,**缺主动 HR 能力**: 智能体不主动建议("建议给 X 候选人发 offer") |
| **3 双向匹配** | 🟡 | TwoWay + 解释 + 权重 | 算法有,**但实际命中率?** |

### 关键洞察

> **16 项需求 100% 形式覆盖 ≠ 做透**。甲方要的是"智能化体验",不是"功能存在"。
> **v8.0 必须把"功能有"变成"真好用"**。

## 维度 2: 服务开关 (用户特别要求)

### 需求

> "增加发布前服务开关(是否加入服务)的功能"

### 现状

v3.0 加了 Feature Flag,**但粒度太细**(灰度发布用),缺**服务级别开关**:

- 缺: admin 关闭某个服务,所有用户看不到
- 缺: 服务对内可用 / 对外关闭
- 缺: 按 org / 按 plan 启用服务
- 缺: 服务依赖关系

### 期望的能力

| 能力 | 说明 |
|---|---|
| **服务级开关** | 每个功能一个开关 (clarifier, career_planner, etc.) |
| **发布前灰度** | 新功能发布前先关,准备 OK 再开 |
| **按 plan 启用** | Free plan 不能用 AI 模拟面试官,Pro+ 可以 |
| **按 org 启用** | 某些企业可以单独启用/关闭 |
| **按角色启用** | HR 可用,求职者不可用 |
| **紧急回滚** | 1 键关闭某服务 |
| **依赖管理** | B 服务依赖 A 服务,A 关闭时 B 自动隐藏 |
| **服务目录** | 公开的 /status/services 页面 |

### 商业价值

| 维度 | 价值 |
|---|---|
| **发布安全** | 新功能发布前可立即关闭 |
| **商业化** | 高级服务按 plan 计费 |
| **灰度** | 内部 dogfood → 公开 |
| **运维** | 服务故障立即关闭,不影响其他 |

### 与现有 Feature Flag 的区别

| 维度 | Feature Flag (v3.0) | Service Toggle (v8.0) |
|---|---|---|
| 粒度 | 功能点 (per request) | 服务整体 (per service) |
| 用途 | A/B 测试 / 灰度 | 发布控制 / 商业化 |
| 维度 | 百分比 / 用户 ID | plan / org / role |
| 持久化 | 临时 | 长期 |
| 依赖 | 无 | 强(服务依赖) |

### 真实缺口

虽然加新"技术"容易,但加新"业务能力"或"甲方需求"还有难度:

1. **需求 → Agent → UI 全链路打通慢**(没有"需求模板")
2. **A/B 测试业务流程 弱**(虽然有 v3.0 算法 A/B,但业务流程 A/B 缺)
3. **用户反馈收集 弱**(没有统一的反馈入口,各种 NPS 散落)
4. **数据驱动决策 弱**(没 BI 工具帮助看哪些需求做得好)

## 维度 4: 16 项需求做透的具体差距

### 关键问题

虽然 v1-v7 加了 100+ 个功能,但**用户体验上**求职者/HR 是否真的"感觉到"智能体是朋友/HR?

#### 求职者侧差距

| 需求 | "做透" 还需要 |
|---|---|
| 1.1 朋友 | 主动生日/节日祝福 / 私聊风格个性化 / 求职空窗期关怀 |
| 1.2 评价 | 行业垂直评价(对 AI 产品经理 vs 后端) / 建议可执行性评估 |
| 1.3 频繁 | 智能体主动 push (今天状态如何?这周面试安排) |
| 1.4 情绪 | 心情差时主动温暖对话 / HR 自动关怀 / 推荐减压资源 |
| 1.5 澄清 | "我理解的你"展示给用户确认 / 需求准确度评分 |
| 1.6 规划 | 计划执行追踪 (3 月 15 学完 X) / 计划调整建议 |

#### 用人单位侧差距

| 需求 | "做透" 还需要 |
|---|---|
| 2.1 真诚 HR | 模板化回复 → 个性化语气 / 老板偏好深度学习 |
| 2.2 验证 | 假资质 AI 检测 / 过期提醒 / 多重交叉验证 |
| 2.3 战略 | 战略对招聘的影响分析 / 战略更新通知 |
| 2.4 偏见 | 老板"看到"偏见后果(招聘效果下降数据) / 替代话术 push |
| 2.5 JD | JD 营销化 (引人入胜) / SEO 优化 |
| 2.6 制度 | "AI 解释制度" (用人话解释) / 求职者提问 |
| 2.7 多方 | notification 触达 / @mention 提醒 / 沉默话题激活 |
| 2.8 澄清 | 共识度算法细化 / 模糊共识 vs 强共识 |
| 2.9 主动 HR | 智能体主动建议 (该给 X 发 offer 了) |
| 3 匹配 | 实际命中率统计 / HR 反馈循环 |
