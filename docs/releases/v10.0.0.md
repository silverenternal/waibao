> **Tag:** `v10.0.0`
> **Date:** 2026-07-15
> **Theme:** 把 v9.x 的产品形态做硬——强契约 Agent Gateway、数据库级多租户、PII/KMS 加密、三域隐私合规 (GDPR / PIPL / CCPA)、以及把 AI 子系统从 mock 推到真实模型。

## Breaking Changes

- **Agent 调用必经 Gateway:** `agents/gateway.py` 现在是唯一入口,绕过它直接调用 agent 内部函数不再受契约/PII 守卫保护。业务侧请迁移到 `AgentGateway.invoke(...)`。
- **错误信封统一:** API 错误响应统一走 `services/platform/errors.py` 的标准信封结构。依赖旧 ad-hoc 错误 JSON 的客户端需对齐新 schema。
- **PII 字段加密:** email / phone / 身份号等 PII 写入前 AES-GCM 加密,`ENCRYPTION_KEY` (KMS/Doppler) 为必填——缺失即 fail-fast。
- **RLS 强制:** 所有业务表 `USING` + `WITH CHECK` 双策略;不带 `tenant_id` 的写入会被数据库拒绝。

## Upgrade Notes

1. **环境变量:** 新增 `ENCRYPTION_KEY`、KMS 配置、PIPL 跨境声明开关——见 `.env` 模板更新。
2. **数据库迁移:** 执行新增的 RLS / 索引 / 分区 / 全文检索迁移脚本。
3. **EventBus:** 升级到 Redis Streams 后端,部署需提供 Redis ≥ 7 + 配置 DLQ。
4. **前端:** PrivacyRightsPanel 区域参数 (`region: EU | CN | CA | GLOBAL`) 决定渲染 GDPR / PIPL / CCPA 控件。
