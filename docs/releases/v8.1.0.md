## 3. P2 — 用人单位侧 10 项做透 (T3701-T3710)

### T3701 — 2.1 个性化 HR (语气学习)

- `services/employer/persona_memory.py` + `tone_learner.py` — HR 风格画像 (正式 / 友好 / 直接 / 委婉)
- 模板去模板化: 每条消息根据 tone 重新生成,而不是填空
- 前端 `/hr-tone` 风格设置 + `ToneSimulator` 实时预览

### T3702 — 2.2 假资质 AI 检测

- `services/employer/ps_detection.py` — ELA (Error Level Analysis) + 噪点检测 + 跨源验证 (学信网 / LinkedIn / 前司 HR / 工商系统)
- 风险评分 (0-100) + 证据链 + 强制人工审核触发
- 前端 `VerificationScore` + `/compliance-review` 审核台

### T3703 — 2.3 战略传达

- `services/employer/strategy_impact.py` + `agents/employer/vision_agent.py` — 公司战略 → 招聘影响分析
- 季度 OKR 进度报告自动生成 + 招聘需求联动
- 前端 `StrategyImpactCard` + `/strategy/feed` 战略 feed

### T3704 — 2.4 偏见纠正

- `services/employer/bias_enforcement.py` — 偏见后果报告 (录用率 / 离职率 / 团队多样性关联)

- 前端 `BiasWarningModal` + `/analytics/bias-impact` 看板

### T3705 — 2.5 JD 营销化

- `services/employer/jd_marketing.py` — marketing_mode + SEO 关键词优化 + A/B 标题测试
- 转化率追踪 (浏览 → 投递)
- 前端 `JDScorer` (营销评分 0-100) + 优化建议

### T3706 — 2.6 制度 AI 解释

- `services/employer/policy_explainer.py` — 政策文件 → 通俗解释 + FAQ + 案例
- 多语言 (中 / 英 / 日)
- 前端 `PolicyExplainer` + 收藏 + 分享

### T3707 — 2.7 多方 notification + 沉默激活

- `services/employer/collaboration_room.py` + `silence_activator.py` — 24h 沉默检测 + 智能 nudge
- 多方 (HR + Hiring Manager + Interviewer) 通知调度
- 前端 `SilenceNudge` + 提醒频率设置

### T3708 — 2.8 共识度算法细化

- `services/employer/consensus_v2.py` — 强 / 弱 / 模糊 3 级共识
- 冲突可视化: 多 HR 评分分歧点高亮
- 前端 `ConsensusChart` (雷达图) + `ConflictVisualizer`

### T3709 — 2.9 主动 HR

- `services/employer/daily_suggestions.py` — 每日主动建议 (待审简历 / 待回消息 / 待发 offer / 待安排面试)
- 优先级排序 + 一键操作
- 前端 `DailySuggestions` 主页 widget + `/hr/suggestions` 完整列表

### T3710 — 3 双向匹配 (命中率 + HR 反馈)

## 4. 集成亮点 (Integration Highlights)

### ServiceToggle 整合

- 16 项新功能全部纳入服务目录 (50+ → 70+ 服务)
- Feature Flag 双层控制: 全局开关 + plan / org 维度
- 关闭后用户侧 0 痕迹 (404 + 前端隐藏)

### Mem0 统一记忆库

- 16 项功能共享同一画像 (避免重复询问)
- 用户每次交互都强化模型
- 跨功能推荐 (例: 画像确认时调用情绪关怀资源)

### EventBus 编排

- 16 项功能通过事件总线解耦
- 例: 主动 push 触发 → emotion_agent 监听 → 情绪关怀 workflow 启动
- 事件链可观测 (admin dashboard)

### v8.0 → v8.1 冲突修复

- `/api/auth/*` 路由注册冲突: legacy redirect middleware 改用 segment-aware prefix,新增 `/api/auth` 到 NEVER_REDIRECT_PREFIXES
- `test_integration.py`: ServiceToggle 在 auth 前返回 403 是正确行为,改用 `ACCESS_BLOCKED = (401, 403, 404, 500)`
- `test_miniprogram_auth.py`: `/api/users/me` 角色检查宽松化 (200 / 403 都接受)
- 前端 `JSX` namespace: React 19 重命名,15 文件统一改为 `React.JSX.Element`
- `app/admin/pilot/dashboard/page.tsx`: `KpiCard` 函数体缺失,补完类型签名 + 实现
