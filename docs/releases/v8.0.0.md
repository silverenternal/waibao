# waibao v8.0.0 — Depth · Control · Data-Driven

> **发布日期**: 2026-07-13 · **代号**: "Depth + Service Toggle + Real DAU"
> **上一版**: [v7.0.0](./v7.0.0.md) (2026-07-13) · **覆盖**: 16 项做透 + 服务开关 (50+ 服务) + 真实日活数据驱动
> **配套**: [SERVICE_TOGGLE_SPEC.md](../SERVICE_TOGGLE_SPEC.md) · [DEPTH_PLAN.md](../DEPTH_PLAN.md) · [VENDOR_SELECTION.md](../VENDOR_SELECTION.md) · [PRIVATE_DEPLOYMENT.md](../PRIVATE_DEPLOYMENT.md)

v8.0 是 waibao 从「**功能铺开**」走向「**功能做透 + 服务可控 + 数据驱动**」的关键一步。围绕三件事:

1. **服务开关 (Service Toggle)** — 50+ 服务可一键关/开,按 plan / org / role 多维度控制,1 键回滚,服务依赖管理
2. **16 项需求做透** — 不再加新功能,深耕现有功能的体验/质量/效果,让甲方要求 100% 落地
3. **真实日活 + 数据驱动** — 5+ 中型企业真实使用,自动周报 + 异常告警 + 用户行为分析

---

## 一行 TL;DR

> **v7 是 SaaS+AI 平台,v8 是平台变得「可控 + 可观测 + 可增长」**。所有服务可关可开,所有反馈自动归类,所有数据自动洞察。

## 2. P0 — 服务开关 (Service Toggle - 发布前控制)

### T3501 — 服务开关核心

- `services/platform/service_toggle.py` — 50+ 服务注册/启用/禁用/回滚
- 多维度控制: 全局 / plan / org / role 4 层
- 60s Redis cache (无 Redis 时降级 in-memory LRU)
- EventBus 事件: `service.changed` / `service.disabled` / `service.rolled_back`
- 持久化审计: `service_audit` 表记录所有变更

### T3502 — 公开服务目录 + 服务订阅

- 用户侧 catalog UI (`/admin/services`) — 显示每个服务的状态/价格/依赖
- 自助订阅 (Starter/Growth/Enterprise 3 档)
- 1 键回滚到任意历史状态

### T3503 — 与 Feature Flag / ConfigCenter 整合

- Service Toggle 与 Feature Flag 双层控制
- ConfigCenter 集中配置 + 热更新
- 所有 65+ 关键 API 加 `Depends(check_service_access)` 守卫

### T3506 — 前端 FeatureGate 包装组件

- `<FeatureGate service="ai_interview">` 组件级隐藏
- Hook `useServiceAccess(name)` 编程式判断
- 关闭后用户侧 0 痕迹 (404 + UI 隐藏)

> **测试**: 服务开关 50+, Feature Access 30+, 公开服务目录 20+, 端到端 10+

## 4. P1 — 数据驱动 (T3901)

### T3901a — 自动周报 (`auto_report.py`)

- 每周一 09:00 自动生成 (scheduler hook)
- 内容: 日活 / 关键功能使用 / **16 项需求使用率** / 异常点
- 格式: PDF (reportlab) / DOCX (python-docx) / TXT fallback
- 邮件给老板 / HRBP (复用 v6.0 notify dispatcher)

### T3901b — 异常检测 (`anomaly_detector.py`)

- 4 类异常:
  - 匹配率突降 (>20%)
  - 用户活跃度突降 (>15%)
  - 工单积压 (>50)
  - 错误率突增 (>5pp)
- 严重程度: critical / warning / info
- 实时告警 (钉钉/飞书/IM/Webhook/SMTP)
- 环境变量可调阈值

### T3901c — 用户行为分析

- 3 类洞察:
  - **popular** (高使用率)
  - **low_usage** (使用率低)
  - **abandoned** (>7 天无调用)
- 自动给 PM 推送下架/重设计建议

### T3901d — Admin Insights Dashboard

- `/admin/insights` 页面
- 最新周报 + 异常 + 行为洞察 三合一
- 一键触发检测 / 生成周报

> **API**:
> - `GET  /api/insights/weekly/latest`
> - `POST /api/insights/cycle` (run detector)
> - `POST /api/insights/weekly/generate` (生成 PDF)
> - `GET  /api/insights/anomalies`
> - `GET  /api/insights/behavior`

> **测试**: auto_report 36, anomaly_detector 43, insights 15 = **94 tests**

## 5. P1 — 用户反馈统一入口 (T3902)

### T3902a — `FeedbackWidget` (前端)

- 全应用通用浮动气泡 (右下角)
- 类型: 评分 (5 星) / Bug / 建议 / 体验 / 性能
- 自动带 context: 当前页 / 用户 / tenant / UA / viewport
- 内联 (inline) 模式可嵌入任意页
- 6 个状态机: closed → select → form → thanks

### T3902b — `feedback_v2` 后端

- `POST /api/feedback/v2` — 收集
- 自动归类 (LLM 启发式): bug / feature / experience / performance / other
- 优先级评分: critical / high / medium / low
- 鉴权: 提交需要登录; admin 才能 list/trend/status

### T3902c — Admin Feedback Dashboard

- `/admin/feedback` 页面
- 5 类筛选: 类型 / 优先级 / 状态 / 全文搜索
- 14 天趋势图 (按日聚合 + critical/high 高亮)
- 一键更新状态 (open/triaged/in_progress/resolved/closed)

> **测试**: 40 tests (含启发式归类、优先级、auth、trend)
