## 🎯 头条

**前端整体重做达到企业级。** 在 v8.1 16 项做透 + 服务开关 + 数据驱动的基础上,前端
按"一个主架构 + 五类模式库"的策略重构:

| 借鉴              | 用在哪                              |
|-------------------|-------------------------------------|
| **Refine**        | admin 资源 / 数据提供器 / 权限       |
| **shadcn-admin**  | mothership 后台壳 / 命令面板 / 表格  |
| **Tremor**        | analytics dashboard                 |
| **Open WebUI**    | jobseeker 对话 / 流式 / 主动 push    |
| **OpenResume**    | 简历上传 / 解析 / 卡片化呈现         |

详见 [`docs/FRONTEND_SELECTION.md`](../FRONTEND_SELECTION.md)。

## 🛠️ 修复 (v8 → v9)

- `Service.from_dict` 过滤 `__missing__` sentinel — 修复 `get_service` 缓存命中失败
- `useEventBus` 增加可选 topics + `subscribe()` API,替代散落的 `useEventBus?.()`
- `useServiceToggle` 暴露 `useServiceDecision` 给需要完整 decision 的 UI
- `useFeatureFlag` 改用新的 `useEventBus` API
- 所有使用 `next/dynamic({ ssr: false })` 的页面加 `"use client"`
- `vitest.config.ts` 移除 Vitest 4 已废弃的 `environmentMatchGlobs`
- `DropdownMenu` 移除 `@radix-ui` 的 `asChild` 用法 (Base UI 不支持)
- `lib/theme.ts` 移除重复 `BrandingBundle` 导出
- `lib/auth-sso.ts` 兼容 NextAuth v5 beta 的 `NextAuthConfig` 类型
- `frontend/.env.local` placeholder 让 `next build` 在无 Supabase 凭证下也能跑通
- `_FakeTable.update/delete` 延迟到 `.execute()`,让 `.eq().update().execute()`
  的链式调用能正确过滤行 (测试桩 bug)
