## 🔴 关键差距 (合同 P0)

### 1. 本地 LLM — "完全不允许离开甲方环境"

**现状**: 所有 LLM 调用走外网 (OpenAI/Anthropic/DeepSeek/智谱/通义/Kimi)
**要求**: 第三方大模型数据**完全不允许离开甲方环境**
**方案**: 集成 **Ollama** 本地 LLM (Qwen2.5-7B / GLM-4-9B)
**工作量**: 2-3 天 (Ollama provider + docker-compose + 测试)

### 2. 招聘市场模型 — "人才来了存储，企业来了存储，匹配，推送"

**现状**: 分散的 Agent 功能,没有统一的「市场」UX
**要求**: 招聘市场 — 人才入库 → 企业入库 → 匹配 → 推送人才给企业
**方案**: 新建 **市场首页** + **人才池** + **岗位池** + **匹配推送** 流程
**工作量**: 3-4 天

### 3. 匹配重写 — 硬条件过滤

**现状**: harmonic_score + 向量语义匹配 + 5 维度评分
**要求**: 只按硬条件自动过滤
  - 技能+岗位职责: 硬条件 (必须满足)
  - 学历+证书: 硬条件 (必须满足)
  - 薪资+城市+工作时间: 高优先级
  - 到岗时间+求职意愿: 高优先级
  - 工作/行业经历: **不使用**
**方案**: 新建 **硬条件过滤器** (structured filter only, 不做语义)
**工作量**: 2 天

### 4. 匹配结果展示

**要求**: 匹配分数 + 匹配理由 + 能力缺口 + 风险提示
**现状**: explainer 有但太复杂
**方案**: 简化匹配卡 (4 个 section: 分数/理由/缺口/风险)
**工作量**: 1-2 天

### 5. 不淘汰候选人

**要求**: AI 不淘汰,只提供建议
**现状**: 匹配算法有"淘汰"逻辑 (score < threshold)
**方案**: 删除淘汰逻辑,改为"排序 + 标记"
**工作量**: 0.5 天

### 6. 推送人才给企业

**要求**: 匹配成功后把人才资料推送给企业
**现状**: 没有"推送"概念
**方案**: 匹配成功后创建"推荐记录"(含完整简历+联系方式),企业可查看/下载
**工作量**: 2 天
