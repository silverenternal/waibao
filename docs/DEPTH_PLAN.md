# 16 项需求做透 — Depth Plan (v8.0)

> Status: **in progress** (v8.0 release scope)
> Owners: @product-team + @platform-team
> Last updated: 2026-07-13

## 1. Background

v1-v7 是「功能铺开」: 16 项甲方需求 100% 形式覆盖. 但 v7 audit 发现:

> 「功能有」≠「做透」

- ProfileCard 是数据展示, 不是知心朋友
- 智能体不主动 push (频繁互动缺)
- 心情差时智能体不主动关怀
- 画像生成后求职者无法确认
- 规划生成后没执行追踪
- HR 偏好存了但模板化回复
- 假资质鉴别弱
- 老板看不到偏见后果
- JD 撰写缺营销化
- 制度缺 AI 解释
- HR 不主动建议

v8.0 战略转变: **不再加新功能, 深耕现有功能的体验/质量/效果**.

## 2. 16 项做透 — Detailed Plan

### 2.1 求职者侧 (1.1-1.6)

#### 1.1 ProfileCard 知心朋友
- **现状 (v7)**: 数据展示 (技能/经验/项目)
- **目标 (v8)**: 情感化对话 + 知心语气
- **关键改动**:
  - 文案风格迁移 (冰冷 JSON → 温暖叙述)
  - 「你最近想做点什么?」开场
  - 根据对话历史主动回顾
- **验收**: NPS 提升 ≥ 5pt

#### 1.2 主动 push 频繁互动
- **现状**: 仅系统通知
- **目标**: 基于情绪/节假日的智能 push
- **关键改动**:
  - 情绪检测触发关心型 push
  - 节日/纪念日触发回忆型 push
  - 求职进度卡点触发建议型 push
  - 频控: 每天 ≤ 3 条
- **验收**: Push 打开率 ≥ 40%

#### 1.3 画像生成后求职者确认
- **现状**: 生成即终态
- **目标**: 主动询问确认 / 修改
- **关键改动**:
  - 画像生成后 5 分钟内 push
  - 「这是不是你?」+ 修改入口
  - 修改后重算匹配
- **验收**: 画像修改率 ≥ 30%

#### 1.4 规划生成后执行追踪
- **现状**: 仅展示
- **目标**: 每步追踪 + 进度可视化
- **关键改动**:
  - 进度条 + 阶段完成度
  - 逾期提醒
  - 阶段奖励 (勋章/称号)
- **验收**: 30 日留存率 ≥ 60%

#### 1.5 心情差时智能体关怀
- **现状**: 无
- **目标**: NLP 情绪检测 + 主动问候
- **关键改动**:
  - 输入文本情绪打分
  - 负面情绪触发关心对话
  - 极端情绪触发人工介入
- **验收**: 用户对关怀对话满意度 ≥ 80%

#### 1.6 HR 偏好 + 模板化回复
- **现状**: 仅存储
- **目标**: 模板化回复 + 一键应用
- **关键改动**:
  - HR 偏好 → 智能体回复模板
  - 一键应用 + 二次编辑
  - 学习 HR 历史回复风格
- **验收**: 模板使用率 ≥ 70%

### 2.2 用人单位侧 (2.1-2.9)

#### 2.1 JD 营销化撰写
- **现状**: 模板填空
- **目标**: 营销化 + 竞品对比 + A/B 标题
- **关键改动**:
  - 同岗位 3 个标题方案
  - 薪资区间建议 (基于市场)
  - 公司亮点自动植入
  - A/B 测试自动跑
- **验收**: JD 浏览 → 投递率 ≥ 8%

#### 2.2 假资质鉴别
- **现状**: 关键词
- **目标**: 跨平台交叉验证
- **关键改动**:
  - 学信网 / LinkedIn / 前司 HR 交叉验证
  - 证书 OCR + 数据库比对
  - 风险评分 (0-100)
- **验收**: 假资质识别率 ≥ 90%

#### 2.3 AI 模拟面试官
- **现状**: 单人格
- **目标**: 5 人格 + 5 阶段 + 情绪感知
- **关键改动**:
  - 友好 / 严格 / 技术 / 压力 / 文化 5 人格
  - 5 阶段: 寒暄 → 履历 → 技术 → 行为 → 反问
  - 候选人情绪感知 + 调整节奏
- **验收**: 候选人完成率 ≥ 75%

#### 2.4 Offer 真实业务
- **现状**: 模板
- **目标**: 法律审查 + 谈判模拟 + 谈薪助手
- **关键改动**:
  - Offer 内容法律审查 (劳动法自动比对)
  - 候选人谈薪模拟 (3 轮)
  - HR 接受 / 还价建议
- **验收**: Offer 接受率 ≥ 65%

#### 2.5 试用期跟踪
- **现状**: 30 天
- **目标**: 90 天分阶段 + 风险预警
- **关键改动**:
  - 7-30-90 三阶段目标拆解
  - 每周 OKR 进度自动采集
  - 风险评分 (OKR 进度 + 1on1 反馈 + 同事评价)
- **验收**: 试用期通过率 ≥ 90%

#### 2.6 内部推荐
- **现状**: 链接
- **目标**: 奖励阶梯 + 实时排行 + 自动对账
- **关键改动**:
  - 内推奖励阶梯 (面试 500 / 入职 5000 / 转正 20000)
  - 实时排行 (周/月/季)
  - 财务自动对账
- **验收**: 内推占比 ≥ 30%

#### 2.7 沉睡激活
- **现状**: 弱
- **目标**: 3 维度画像 + 智能触达
- **关键改动**:
  - 3 维度: 求职意愿 / 经济压力 / 行业热度
  - 4 通道触达: 短信 / 推送 / 邮件 / IM
  - 转化漏斗追踪
- **验收**: 沉睡激活率 ≥ 15%

#### 2.8 偏见检测
- **现状**: 弱
- **目标**: 后果可视化
- **关键改动**:
  - 老板侧 dashboard: 录用率 / 离职率 / 多样性指数
  - 偏见与业务结果关联分析
  - 整改建议
- **验收**: 偏见评分降低 ≥ 30%

#### 2.9 制度 AI 解释
- **现状**: 弱
- **目标**: 条文 + 案例 + FAQ
- **关键改动**:
  - 公司制度全文索引
  - 「这个什么意思?」自然语言解释
  - 相关案例 + FAQ 推送
- **验收**: 员工满意度 ≥ 80%

### 2.3 综合 (3)

#### 3 战略传达 + 主动 HR 建议
- **现状**: 无
- **目标**: 季度自动 OKR 报告 + 主动建议
- **关键改动**:
  - OKR 进度自动采集
  - 季度战略报告自动生成
  - HR 主动建议 (招聘节奏 / 团队配置 / 风险预警)
- **验收**: 老板使用率 ≥ 70%

## 3. KPI 跟踪

- 每周 16 项使用率自动生成 (T3901 auto_report)
- 每月复盘做透进度
- 季度对比 v7 基线

## 4. 资源

- 1 PM + 1 Tech Lead + 3 开发
- 6 周 (P1 + P2 阶段)
- 与服务开关 (P0) 并行

## 5. 风险

- 「做透」主观性强, 需要持续用户反馈
- 部分需求 (如假资质鉴别) 依赖第三方数据
- LLM 成本上升 → 用 v7.0 prompt v2 优化
